█ 二、测试程序结构
════════════════════════════════════════════════════════════════════════════

  ChassisControlTester 类包含以下测试：

  测试 1: control_chassis_to_center() - 目标中心对齐
  ──────────────────────────────────────
    场景1: 目标在右侧 (+100px) 
      预期: 机器人左转对齐目标
      验证: 获取位置确认转向成功

    场景2: 目标在左侧 (-150px)
      预期: 机器人右转对齐目标
      验证: 获取位置确认转向成功

    场景3: 目标已在中心
      预期: 不转向（在公差范围内）
      验证: 机器人保持静止

    场景4: 大偏差 (+250px)
      预期: 机器人进行大幅转向
      验证: 角度计算正确性


  测试 2: control_chassis_forward() - 直线前进
  ──────────────────────────────────────
    场景1: 前进 0.5m
      预期: 机器人前进 0.5m
      验证: 位置变化 ≈ 0.5m

    场景2: 前进 1.0m
      预期: 机器人前进 1.0m
      验证: 位置变化 ≈ 1.0m

    场景3: 小距离前进 (0.2m)
      预期: 机器人精确前进 0.2m
      验证: 位置精度要求高


  测试 3: control_turn_around() - 原地转向
  ──────────────────────────────────────
    场景1: 转向 90°
      预期: 机器人原地转向 90°
      验证: 位置不变，角度变化 ≈ 90°

    场景2: 转向 -45°
      预期: 机器人逆时针转向 45°
      验证: 位置不变，角度变化 ≈ 45°

    场景3: 转向 180°（掉头）
      预期: 机器人掉头
      验证: 位置不变，角度变化 ≈ 180°


  测试 4: 连续操作序列测试
  ──────────────────────────────────────
    序列1: 对齐 → 前进 → 转向
      3 个步骤的组合测试
      
    序列2: 前进 → 对齐 → 前进 → 转向（复杂序列）
      4 个步骤的组合测试
      验证: 状态转移的连贯性


  测试 5: 压力测试 - 重复调用
  ──────────────────────────────────────
    压力测试1: 连续转向 10 次 (每次 10°)
      目标: 测试转向函数的稳定性
      成功标准: ≥ 80% 成功率

    压力测试2: 连续对齐 5 次（不同目标位置）
      目标: 测试对齐函数的可靠性
      成功标准: ≥ 80% 成功率


  测试 6: 错误恢复和边界情况
  ──────────────────────────────────────
    边界测试1: 无效 bbox (超出范围)
      预期: 优雅处理或异常捕获
      
    边界测试2: 0 距离前进
      预期: 快速返回成功
      
    边界测试3: 0 角度转向
      预期: 快速返回成功

█ 五、测试参数配置
════════════════════════════════════════════════════════════════════════════

  在测试程序中可以修改的参数：

  1. 等待时间：
     ──────────
     self.SHORT_WAIT = 3      # 短操作后的等待时间（秒）
     self.MID_WAIT = 5        # 中等操作后的等待时间
     self.LONG_WAIT = 10      # 长操作后的等待时间

  2. 前进距离：
     ──────────
     test_chassis_forward() 中的 distance 参数
     
     建议值:
       • 小距离: 0.2m - 0.5m
       • 中等: 0.5m - 1.0m
       • 大距离: 1.0m - 2.0m（需要空间）

  3. 转向角度：
     ──────────
     test_turn_around() 中的 angle_deg 参数
     
     建议值:
       • 小角度: 10° - 45°
       • 中等: 45° - 90°
       • 大角度: 90° - 180°

  4. 目标框位置：
     ────────────
     test_chassis_to_center() 中的 bbox 参数
     格式: [x1, y1, x2, y2]
     范围: 0-1000 (1000×1000 图像)
     
     建议值:
       • 左侧: [100-300, 400, 300-500, 600]
       • 中心: [400-500, 400, 500-600, 600]
       • 右侧: [600-800, 400, 800-1000, 600]


█ 六、常见问题排查
════════════════════════════════════════════════════════════════════════════

  问题 1: "连接被拒绝" (Connection refused)
  ──────────────────────────────────────────
  原因: AGV 服务未启动或地址错误
  解决:
    1. 检查 AGV 服务是否启动: http://192.168.10.10:31001
    2. 检查网络连接
    3. 确认 IP 地址正确
    4. 使用 ping 测试: ping 192.168.10.10

  问题 2: "超时" (Timeout)
  ─────────────────────────
  原因: 底盘命令执行时间过长
  解决:
    1. 增加 poll_interval 和 timeout 参数
    2. 检查底盘是否卡住
    3. 查看底盘日志
    4. 检查环境障碍物

  问题 3: "位置不变化"
  ──────────────────
  原因: get_current_pose() 可能未正确更新
  解决:
    1. 检查导航模块的状态监控线程
    2. 增加等待时间
    3. 手动验证底盘是否移动

  问题 4: "精度差"
  ────────────────
  原因: 可能需要底盘标定或参数调整
  解决:
    1. 检查底盘电机参数
    2. 进行轮子标定
    3. 调整控制算法参数
    4. 检查地面摩擦力

  问题 5: "某个测试总是失败"
  ────────────────────────
  原因: 可能是硬件问题或参数不合适
  解决:
    1. 单独测试该功能
    2. 调整参数（距离/角度更小）
    3. 检查底盘硬件
    4. 查看底盘日志
