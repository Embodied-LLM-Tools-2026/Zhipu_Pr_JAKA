🌌 AUREKA 项目方案（V2）—— 从任务驱动到技能进化的闭环系统
🎯 项目目标（一句话版）
构建一个让 LLM 能够在仿真环境中持续迭代、扩展、压缩并正交化机器人动作原语（Skill Primitives）的系统，使机器人能够通过任务难度递增来自举出“完备且高效”的动作空间。
🧩 一、核心思想：将“动作原语”视为工具库自动扩张的过程
你给的图展示了一个核心概念：
复杂任务不直接解决，而是促使 Agent 先构建必要的工具链，然后利用工具链不断升级自身。
应用到机器人：
任务集合不是用来直接完成的
任务集合本质上是一个压力源（pressure source）
用来驱动 LLM 自动扩展动作原语（skills）
从而自动形成：
“会接近物体”
“会抓取”
“会对齐”
“会旋转”
“会拉/推”
“会调姿”
…
最终形成动作基底（action basis）。
这是具身智能领域里非常缺的一块：
让机器人自动学会需要哪些技能，而不是靠人类硬编码。
🏗️ 二、总体架构设计（Architecture Overview）
系统由三个模块构成：
1）Generator（LLM / Code Synthesizer）
根据任务失败反馈、现有技能库、机器人 API 文档：
创建新的技能
修复低效技能
合并冗余技能
删除 useless 技能
输出：一组新的 SkillPrimitive 代码函数
2）Executor（Environment + Planner）
在仿真中执行任务：
根据技能库构建 task solver（随机、行为树、MCTS、RL……都可）
测试技能在不同任务中的表现
收集所有轨迹
输出指标：
success rate
state change vector
effect embedding
coverage
redundancy
error type counts
skill usage contribution
3）Reflector（Evaluator / Critic）
从轨迹分析中得出自然语言反馈：
哪些技能无用
哪些技能效果重复
哪些功能缺失
哪些可以合并/重写
哪些任务失败原因来自技能不足
哪些任务需要新增 primitive
输出：结构化自然语言反馈（给 LLM）
🧬 三、系统闭环（Core Loop）
对照你给的图，我将 AUREKA 的闭环对应描述：
Step 1：给定一类任务分布（任务课程 Curriculum）
例如：
Level 1：移动到目标点
Level 2：接近物体
Level 3：抓取
Level 4：搬运
Level 5：置放
Level 6：开抽屉
Level 7：旋转拧盖
Level 8：插入对齐
重点：
任务按能力需求逐步增加复杂度（Curriculum Learning）。
Step 2：根据任务集合自动进化技能库（Toolchain Synthesis）
LLM 先生成基础技能：
move_delta
approach
open_gripper
close_gripper
随着环境失败，逐步扩展：
orientation_align
pinch_grasp
compliant_push
hook_pull
pivot_turn
这些就是“工具函数”。
Step 3：对简单任务推导与验证（Bootstrapping）
与图中一致：
先用最简单任务检验技能库
再往难题推进
如果失败→扩充技能库
Step 4：任务难度递增时不断修订技能库（Tool Evolution）
包括：
新增技能
改写技能
扩展参数
缩减冗余
标准化接口
合并共线技能
修复不稳定控制律
技能库的演化类似：
Round 1：5 个基础技能  
Round 3：增加了 3 个抓取相关技能  
Round 6：删掉重复 push、增加 rotate 和 align  
Round 10：形成一个正交化的基础动作空间  
Step 5：如果工具不够用 → 自动扩展技能库（Skill Discovery）
这是关键创新点：
任务分布迫使 LLM 生成新的动作维度 → 形成具身智能的动作符号自动落地。
例如：
任务：开抽屉
失败原因：只能 push，不能 hook
系统自动生成：
hook_pull(angle, force)
任务：旋紧瓶盖
失败原因：不能持续稳定旋转
系统生成：
twist_grasp(speed, torque)
🔍 四、任务定义（Task Specification）
任务输入
物体初始位置
一组指定目标
成功条件（基于传感器与状态）
失败条件（越界、丢物等）
任务输出（Executor）
完成状态
机器人轨迹
技能调用序列
技能失败原因
状态变化向量
📐 五、技能评价指标（Metrics）
结合你给的图的启发，我给你 7 维度指标：
1. Coverage（覆盖率）
技能是否能让末端达到更多的空间区域？
（状态空间探索能力）
2. Utility（有效性）
执行技能后的 L2 位移量（或状态变化量）
3. Success Rate（稳定性）
执行次数中成功比例
4. Redundancy（冗余度）
技能间的效果向量余弦相似度
高冗余 → 删除技能或合并参数
5. Error Pattern（错误模式）
如：越界/碰撞/奇异点/夹爪丢物
6. Task Contribution（任务贡献度）
一个技能对任务的解决贡献分布
（类似 Shapley Value）
7. Missing Capability（缺失能力）
Reflector 从任务失败中抽象能力需求：
例如：
“需要 rotate”
“需要 side-grasp”
“需要 alignment”
“需要 compliant force control”
🧱 六、Skill Primitive DSL 设计
一个动作原语定义为：
@dataclass
class SkillPrimitive:
    name: str
    description: str
    params: Dict[str, str]
    code: str  # Python function
关键要求：
可执行
可分析
可优化
可参数化
可删除/合并
可由 LLM 重写结构
🧪 七、执行器（Executor）设计
三个层次：
Level 1：Random Babbling Planner
最简单：
随机采样技能
-尝试完成任务
适合 Bootstrap 阶段。
Level 2：Heuristic Planner
如：
approach → grasp → lift → move → place
适合中期。
Level 3：RL / MCTS-with-Skills
利用技能作为 Action Abstraction。
适合高级任务。
🧠 八、Reflector（反思器）
从日志中抽象技能库的缺陷：
“动作 push 和 shove 效果重复度 0.92”
“缺少 rotation 能力”
“move_to 在 z 维度收敛太慢”
“grasp 开合角度不足，没有包络抓取”
“skill align 失败，因为缺少 orientation sensing”
这些会被压缩成自然语言反馈给 LLM。
🚀 九、系统运行流程（Master Loop）
for curriculum_level in levels:
    for iteration in range(K):
        run_executor()
        compute_metrics()
        feedback = reflector(metrics)
        skill_library = llm_generate_new_skills(feedback)
最终输出：
一个不断进化的动作库
一个越来越完备的技能空间
一个自动构造技能符号的具身智能系统
🌟 十、项目真正的创新点（非常科研）
⭐1. “动作空间”本身自动进化（Action Space Evolution）
首次提出：
不像 RL、VLA、RT-2、Code-as-Policy 这些工作在固定 action space 上优化策略，这个项目：
把 Action Space 当成优化对象，而非固定前提。
这是一个巨大的 paradigm shift。
⭐2. 技能作为代码函数（Skill as Executable Code）
LLM 不是生成策略，而是生成一整个具身智能的“动作生态系统”。
⭐3. 任务驱动技能扩张（Task-Induced Skill Expansion）
任务不是给机器人做的，是给技能库施压的。
⭐4. 正交技能自压缩（Orthogonality-aware Skill Compression）
自动消除冗余技能，构造一个可泛化、可组合的“动作基底”。
⭐5. 闭环式符号落地（Symbol Grounding via Performance Feedback）
动作描述不靠人类写，而靠真实执行结果来定义。
🌈 十一、结语（一句话总结项目价值）
AUREKA 的目标不是让机器人完成任务，而是让机器人学会“需要什么动作”才能完成任务。
这直接攻克具身智能最核心的符号落地难题
——一个能够自动生长自身动作结构的系统。
