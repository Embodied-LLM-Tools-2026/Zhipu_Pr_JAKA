# 人形机器人项目 - 阶段一 完整文档

## 📋 项目概述

这是一个基于语音控制的人形机器人系统，集成了语音识别、自然语言理解、计算机视觉、机器人控制等多个先进技术模块。项目采用模块化设计，支持实时语音交互、目标检测、精确抓取等功能。

### 🎯 核心功能
- **语音交互**：支持中文语音识别和语音合成
- **智能理解**：基于大语言模型的自然语言理解
- **视觉感知**：实时目标检测和位置定位
- **精确控制**：机器人手臂和手部的精确运动控制
- **动作执行**：预定义动作序列和自定义动作

### 🚀 技术特点
- 模块化架构设计
- 实时语音处理
- 深度学习目标检测
- 机器人运动学控制
- 多模态传感器融合

## 📁 核心模块详解

### 1. 🎤 语音处理模块 (`voice/`)

#### 1.1 模块概述
语音处理模块是整个系统的核心交互接口，负责语音识别、自然语言理解、语音合成和机器人控制指令的生成。

#### 1.2 核心组件

##### `config.py` - 配置管理
- **功能**：集中管理所有配置参数
- **包含内容**：
  - API密钥配置（智谱AI、机器人连接等）
  - 音频参数（采样率、声道数、块大小）
  - 机器人连接参数（IP地址、端口）
  - 唤醒词和取消词配置
  - 常用短语和动作映射

##### `VAD.py` - 语音活动检测
- **功能**：实时检测语音开始和结束
- **技术实现**：
  - `VoiceActivityDetector`：基于能量的基础检测
  - `SileroVAD`：基于深度学习的Silero VAD模型
  - `CircularBuffer`：音频缓冲管理
  - `AdvancedVoiceActivityDetector`：高级VAD算法
- **特点**：
  - 支持多种VAD算法自动降级
  - 实时音频缓冲和预处理
  - 自适应阈值调整

##### `ASR.py` - 语音识别
- **功能**：将语音转换为文字
- **技术**：基于SenseVoice/FunASR
- **主要类**：`SenseVoiceRecognizer`
- **特点**：
  - 支持多种音频格式
  - 实时语音转文字
  - 中文语音识别优化

##### `LLM.py` - 大语言模型接口
- **功能**：自然语言理解和指令解析
- **技术**：基于智谱AI GLM-4
- **主要类**：`LLMInterface`
- **特点**：
  - 自然语言理解
  - 对话生成
  - 机器人指令解析
  - 上下文管理

##### `TTS.py` - 文本转语音
- **功能**：将文本转换为语音输出
- **技术**：基于Edge TTS
- **主要类**：
  - `TextToSpeechEngine`：TTS引擎
  - `StreamingTTSPlayer`：流式播放器
- **特点**：
  - 支持多语言
  - 音频缓存机制
  - 流式播放优化

##### `PinyinMatcher.py` - 拼音匹配
- **功能**：唤醒词检测和命令识别
- **技术**：基于pypinyin的模糊匹配
- **主要类**：`PinyinMatcher`
- **特点**：
  - 唤醒词检测
  - 取消命令识别
  - 拼音变体生成
  - 模糊匹配算法

##### `ActionExecuter_hand.py` - 动作执行器
- **功能**：解析和执行机器人动作指令
- **主要类**：`ActionExecuter`
- **特点**：
  - 动作指令解析
  - 动作队列管理
  - 动作状态跟踪
  - 与机器人控制模块集成

##### `VoiceRobotController.py` - 主控制器
- **功能**：整合所有语音模块的主控制器
- **主要类**：`VoiceRobotController`
- **特点**：
  - 模块整合管理
  - 状态管理（睡眠/唤醒）
  - 命令处理流程
  - 性能监控
  - 错误处理

#### 1.3 语音模块工作流程
```
语音输入 → VAD检测 → ASR识别 → LLM理解 → 动作执行 → TTS反馈
```

### 2. 🤖 机器人动作序列模块 (`action_sequence/`)

#### 2.1 模块概述
动作序列模块定义了机器人的各种预定义动作，包括基础动作、复合动作和自定义动作序列。

#### 2.2 核心组件

##### `execute_action.py` - 动作执行
- **功能**：定义和执行机器人动作
- **主要动作**：
  - `wave()`：挥手动作
  - `bow()`：鞠躬动作
  - `Nod()`：点头动作
- **特点**：
  - 基于X5 API的精确控制
  - 关节角度和位姿控制
  - 动作序列编排

##### `agv_client.py` - AGV客户端
- **功能**：AGV（自动导引车）通信和控制
- **特点**：
  - 网络通信协议
  - 状态监控
  - 指令发送

##### `calibration_action.py` - 标定动作
- **功能**：机器人标定相关动作
- **特点**：
  - 位置标定
  - 姿态标定
  - 精度优化

##### `correct_hand.py` - 手部校正
- **功能**：机器人手部位置和姿态校正
- **特点**：
  - 手部位姿调整
  - 抓取精度优化
  - 动态校正

##### `pour_coffee.py` - 倒咖啡动作
- **功能**：倒咖啡的完整动作序列
- **特点**：
  - 多步骤动作编排
  - 精确控制

##### `PP_gripper.py` - 夹爪控制
- **功能**：夹爪的开合控制
- **特点**：
  - 夹爪状态管理
  - 抓取力度控制
  - 安全保护

##### `PP_hand.py` - 手部控制
- **功能**：机器人手部的综合控制
- **特点**：
  - 手指协调控制
  - 抓取策略
  - 力反馈控制

##### `read_pose.py` - 位姿读取
- **功能**：读取机器人当前位姿信息
- **特点**：
  - 实时位姿监控
  - 数据记录
  - 状态分析

##### `inte_joint.py` - 关节插值
- **功能**：关节运动的平滑插值
- **特点**：
  - 运动轨迹优化

##### `accept_letter.py` - 接收信件动作

- **特点**：

## 🚀 部署和使用

### 环境要求
- **Python版本**：3.10
- **操作系统**：Windows 10/11 或 Linux
- **硬件要求**：
  - 支持CUDA的GPU（推荐）
  - 至少8GB内存
  - 高速网络连接

### 安装步骤

#### 1. 环境准备
```bash
# 创建虚拟环境
conda create --name pr python=3.10
conda activate pr

# 安装基础依赖
pip install -r requirements.txt
```

#### 2. 模型下载
```bash
# 下载VAD模型
python -c "import torch; torch.hub.load('snakers4/silero-vad', 'silero_vad')"

# 设置HuggingFace镜像
export HF_ENDPOINT=https://hf-mirror.com

# 下载语音合成模型
huggingface-cli download --resume-download hexgrad/Kokoro-82M-v1.1-zh --local-dir ./ckpts/kokoro-v1.1
```

#### 3. X5 API安装
```bash
# Windows平台
cd whl
pip install xapi-3.1.11-cp310-cp310-win_amd64.whl

# Linux平台
cd whl
pip install xapi-3.1.11-cp310-cp310-linux_x86_64.whl
```

### 运行方式

#### 主程序运行
```bash
# 运行手部控制版本
python main_hand.py

# 运行语音机器人控制器
python voice/VoiceRobotController.py
```
