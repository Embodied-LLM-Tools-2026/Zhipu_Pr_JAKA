#### 2.1 Affordance函数（SayCan）

**重要程度**：⭐⭐⭐⭐⭐  
**适配程度**：⭐⭐⭐⭐⭐  
**实现状态**：🔴 未实现

**核心思想**：
```
最终决策 = Say（LLM语义评分）× Can（Affordance可行性评分）
```

**为什么重要**：
- 解决LLM"纸上谈兵"问题：LLM可能建议物理上不可行的动作
- 提供物理世界的反馈：让系统知道"能不能做"
- 显著提升任务成功率：SayCan实验显示从32%提升到74%

**代码实现建议**：

```python
class SkillAffordance:
    """技能可行性评估器"""
    
    def __init__(self, world_model):
        self.world = world_model
        self.success_history = {}  # 记录历史成功率
    
    def evaluate(self, skill_name: str, **kwargs) -> float:
        """
        评估技能可行性
        
        Returns:
            可行性分数 (0-1)
        """
        if skill_name == "navigate_to":
            target = kwargs.get("target")
            if target in self.world.known_areas:
                return 0.9
            return 0.1
        
        elif skill_name == "grasp":
            target = kwargs.get("target")
            obj = self.world.objects.get(target)
            if obj and obj.visible and obj.reachable:
                return 0.85
            return 0.05
        
        elif skill_name == "place":
            if self.world.holding_object:
                return 0.9
            return 0.0
        
        return 0.5
```

**与现有代码的结合点**：
- 在 `planner.py` 中集成Affordance评估
- 在 `world_model.py` 中添加状态查询接口
- 在 `executor.py` 中记录执行结果用于学习

#### 2.2 闭环反馈机制（Inner Monologue）

**重要程度**：⭐⭐⭐⭐⭐  
**适配程度**：⭐⭐⭐⭐  
**实现状态**：🔴 未实现

**核心思想**：
```
计划 → 执行 → 反馈 → 反思 → 新计划 → ...
```

**为什么重要**：
- 开环规划无法适应动态环境
- 能够自动处理失败和异常
- 支持人类干预和协作

**三种反馈类型**：

| 反馈类型 | 作用 | 实现方式 |
|---------|------|---------|
| 成功检测 | 判断动作是否成功 | 检查状态变化 |
| 场景描述 | 理解当前环境 | VLM描述 |
| 人类反馈 | 接收外部指导 | 用户输入 |

**代码实现建议**：

```python
class InnerMonologue:
    """内心独白生成器"""
    
    def execute_with_feedback(self, instruction: str):
        while not self.task_completed():
            # 1. 生成计划
            plan = self.llm.plan(instruction, self.scene_description)
            
            # 2. 执行动作
            action = plan[0]
            result = self.executor.execute(action)
            
            # 3. 收集反馈
            success = self.success_detector.detect(action, result)
            scene_desc = self.vlm.describe(self.current_image)
            
            # 4. 生成内心独白
            monologue = self.llm.reflect(
                instruction, action, success, scene_desc
            )
            
            # 5. 调整计划
            if not success:
                plan = self.llm.replan(monologue)
```

```python
class ValueMap3D:
    """3D值图"""
    
    def __init__(self, resolution=0.01):
        self.resolution = resolution
        self.map = np.zeros((100, 100, 100))
    
    def add_affordance(self, position, sigma=0.05):
        """添加可行区域"""
        self._add_gaussian(position, sigma, amplitude=1.0)
    
    def add_constraint(self, position, sigma=0.05):
        """添加约束区域"""
        self._add_gaussian(position, sigma, amplitude=-1.0)
    
    def compose(self, other_map, operation="multiply"):
        """组合值图"""
        if operation == "multiply":
            self.map *= other_map.map
        elif operation == "add":
            self.map += other_map.map

class VoxPoserPlanner:
```

```python
class CodeEngineer:
    """改进版代码工程师"""
    
    FORBIDDEN_OPERATIONS = [
        "os.system", "subprocess", "eval", "exec", "__import__"
    ]
    
    def generate_safe_code(self, task_description: str) -> str:
        # 1. 生成主函数
        main_code = self._generate_main(task_description)
        
        # 2. 识别未定义函数
        undefined = self._find_undefined_functions(main_code)
        
        # 3. 递归生成未定义函数
        for func in undefined:
            func_code = self._generate_function(func)
            main_code += f"\n\n{func_code}"
        
        # 4. 安全检查
        if not self._validate_safety(main_code):
            raise SecurityError("代码包含危险操作")
        
        return main_code
    
    def _validate_safety(self, code: str) -> bool:
        for op in self.FORBIDDEN_OPERATIONS:
            if op in code:
                return False
        return True
```

#### 2.7 动作Token化（RT-2）

**重要程度**：⭐⭐⭐  
**适配程度**：⭐⭐  
**实现状态**：🔴 未实现

**核心思想**：
```
动作 → 离散化 → Token → 与语言Token统一建模
```

**为什么重要**：
- 实现互联网知识到机器人控制的知识迁移
- 获得语义推理能力（如"最小的物体"）
- 展现涌现能力

**实现复杂度**：⚠️ 高（需要训练大型VLA模型）

**简化实现方案**：

```python
class ActionTokenizer:
    """动作Token化器（简化版）"""
    
    def __init__(self, bins=256):
        self.bins = bins
    
    def encode(self, action: np.ndarray) -> List[int]:
        """连续动作 → Token"""
        tokens = []
        for value in action:
            normalized = (value + 1) / 2  # [-1,1] → [0,1]
            token = int(normalized * (self.bins - 1))
            tokens.append(token)
        return tokens
    
    def decode(self, tokens: List[int]) -> np.ndarray:
        """Token → 连续动作"""
        action = []
        for token in tokens:
            normalized = token / (self.bins - 1)
            value = normalized * 2 - 1  # [0,1] → [-1,1]
            action.append(value)
        return np.array(action)
```

**硬件泛化应用**：
- 不同硬件的动作空间可以统一表示
- 支持跨硬件的知识迁移

```python
    """场景可行性评估"""
    
    def evaluate_grasp(self, target: str) -> float:
        """评估抓取可行性"""
        obj = self.world_model.get_object(target)
        
        if not obj:
            return 0.0  # 物体不存在
        
        if not obj.visible:
            return 0.1  # 物体不可见
        
        if obj.distance > self.reach_limit:
            return 0.2  # 物体太远
        
        if obj.size < self.grasp_limit:
            return 0.3  # 物体太小
        
        return 0.9  # 可以抓取
```

```python
class ClosedLoopExecutor:
    """闭环执行器"""
    
    def execute_with_recovery(self, plan: List[Action]):
        for action in plan:
            max_retries = 3
            
            for attempt in range(max_retries):
                result = self.execute(action)
                
                if result.success:
                    break
                
                # 失败恢复
                recovery_plan = self.generate_recovery(action, result.error)
                self.execute_with_recovery(recovery_plan)
```

```python
    def analyze_scene(self, image) -> SceneDescription:

        objects = self.vlm.detect_objects(image)
        
        # 2. 空间关系
        relations = self.vlm.analyze_spatial_relations(image, objects)
        
        # 3. 可操作性分析
        affordances = {}
        for obj in objects:
            affordances[obj.name] = self.analyze_affordance(obj)
        
        return SceneDescription(

        )
```
