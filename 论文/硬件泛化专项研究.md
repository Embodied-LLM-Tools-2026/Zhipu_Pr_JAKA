# 硬件泛化专项研究：前沿技术与实施方案

## 文档概述

本文档针对**硬件泛化**这一核心需求，调研了前沿技术并提出了可行的实施方案。目标是实现：**通过提示词工程，只需填入硬件参数即可生成适配代码**，支持不同机械臂数量、不同底盘类型等硬件配置。

---

## 一、问题定义

### 1.1 当前痛点

| 问题 | 具体表现 | 影响 |
|------|---------|------|
| **硬件耦合严重** | 代码与特定硬件绑定 | 新硬件需要重写大量代码 |
| **配置复杂** | 每个新硬件都需要手动配置 | 接入周期长（数天到数周） |
| **缺乏统一抽象** | 不同硬件API差异大 | 难以复用已有代码 |
| **多硬件协同困难** | 多机械臂、异构底盘难以统一管理 | 系统复杂度高 |

### 1.2 目标场景

```
场景1：单机械臂 + 轮式底盘
├── JAKA Zu7 机械臂
├── AGV 轮式底盘
└── 标准夹爪

场景2：双机械臂 + 轮式底盘
├── 左臂：JAKA Zu7
├── 右臂：JAKA Zu5
├── AGV 轮式底盘
└── 双夹爪

场景3：单机械臂 + 足式底盘
├── JAKA Zu7 机械臂
├── 四足机器人底盘
└── 标准夹爪

场景4：三机械臂 + 移动平台
├── 主臂：JAKA Zu7
├── 辅助臂1/2：JAKA Zu3
├── 移动平台
└── 多夹爪配置
```

## 二、前沿技术调研

### 2.1 跨硬件迁移学习（Cross-Embodiment Transfer）

#### 2.1.1 Open X-Embodiment / RT-X（Google DeepMind, 2024）

**核心贡献**：
- 收集了**22种不同机器人**的数据集
- 提出了**RT-X模型**，实现跨机器人知识迁移
- 证明了**统一数据格式**可以实现跨硬件泛化

**关键技术**：

```
不同机器人 → 统一动作空间表示 → 共享模型 → 迁移到新机器人
```

**对我们的启发**：
- ✅ 设计统一的硬件描述格式
- ✅ 标准化动作空间
- ✅ 建立硬件特征到统一空间的映射

#### 2.1.2 Octo（UC Berkeley, 2024）

**核心贡献**：
- 开源通用机器人操控框架
- 支持**跨形态（cross-embodiment）**迁移
- 仅需**100条数据**即可微调到新硬件

**关键技术**：

```python
# Octo的跨形态迁移机制
class OctoPolicy:
    def __init__(self):
        self.transformer = TransformerPolicy()
        self.embodiment_encoder = EmbodimentEncoder()
    
    def adapt_to_new_hardware(self, hardware_config, demo_data):
        # 1. 编码硬件特征
        embodiment_token = self.embodiment_encoder.encode(hardware_config)
        
        # 2. 微调策略
        self.finetune(demo_data, embodiment_token)
        
        # 3. 返回适配后的策略
        return self
```

**对我们的启发**：
- ✅ 可以设计**硬件特征编码器**
- ✅ 将硬件配置作为条件输入LLM
- ✅ 少量示例即可适配新硬件

#### 2.1.3 Gato（DeepMind, 2022）

**核心贡献**：
- **多形态通用策略**：同一网络可以控制不同形态的机器人
- 统一处理：Atari游戏、图像描述、机器人操作等

**关键技术**：

```
统一Token表示：
- 图像 → Token序列
- 状态 → Token序列
- 动作 → Token序列
- 硬件类型 → 特殊Token
```

**对我们的启发**：
- ✅ 将硬件类型作为上下文信息

#### 2.3.2 Embodiment-Aware Prompting

**核心思想**：将硬件特征作为条件输入LLM

```python
class EmbodimentAwarePrompter:
    """硬件感知的提示词生成器"""
    
    def __init__(self):
        self.hardware_templates = self._load_templates()
    
    def generate_prompt(self, hardware_config: dict) -> str:
        # 1. 提取硬件特征
        features = self._extract_features(hardware_config)
        
        # 2. 选择合适的模板
        template = self._select_template(features)
        
        # 3. 填充参数
        prompt = template.format(**features)
        
        return prompt
    
    def _extract_features(self, config: dict) -> dict:
        """提取硬件特征"""
        return {
            "arm_count": config.get("arms", []).__len__(),
            "arm_types": [arm["type"] for arm in config.get("arms", [])],
            "base_type": config.get("base", {}).get("type", "wheeled"),
            "gripper_types": [g["type"] for g in config.get("grippers", [])],
            "sensor_types": [s["type"] for s in config.get("sensors", [])],
            "dof_total": sum(arm.get("dof", 6) for arm in config.get("arms", [])),
        }
```

### 3.2 硬件特征向量

```python
@dataclass
class HardwareFeatures:
    """硬件特征向量"""
    
    # 机械臂特征
    arm_count: int
    arm_dofs: List[int]  # 每个臂的自由度
    arm_types: List[str]
    total_dof: int
    
    # 底盘特征
    base_type: str  # wheeled, tracked, legged, omni
    is_mobile: bool
    has_odometry: bool
    
    # 末端执行器特征
    gripper_count: int
    gripper_types: List[str]
    
    # 传感器特征
    has_depth_camera: bool
    has_rgb_camera: bool
    has_force_sensor: bool
    
    # 通信特征
    communication_protocols: List[str]  # ros2, tcp, modbus, serial
    
    def to_embedding(self) -> np.ndarray:
        """转换为特征向量"""
        features = [
            self.arm_count,
            self.total_dof,
            1 if self.is_mobile else 0,
            self.gripper_count,
            1 if self.has_depth_camera else 0,
            # ... 更多特征
        ]
        return np.array(features)
    
    def to_description(self) -> str:
        """转换为自然语言描述"""
        desc = f"机器人配置：{self.arm_count}个机械臂"
        if self.is_mobile:
            desc += f"，{self.base_type}底盘"
        desc += f"，{self.gripper_count}个夹爪"
        return desc
```

### 5.1 核心接口定义

```python
# voice/hardware/unified_api.py

from abc import ABC, abstractmethod
from typing import List, Dict, Any, Optional
from dataclasses import dataclass
from enum import Enum

class HardwareType(Enum):
    ARM = "arm"
    BASE = "base"
    GRIPPER = "gripper"
    SENSOR = "sensor"

@dataclass
class HardwareState:
    """硬件状态"""
    name: str
    type: HardwareType
    is_connected: bool
    is_enabled: bool
    error_message: Optional[str] = None
    custom_state: Optional[Dict[str, Any]] = None

class HardwareInterface(ABC):
    """硬件接口基类"""
    
    @abstractmethod
    async def connect(self) -> bool:
        """连接硬件"""
        pass
    
    @abstractmethod
    async def disconnect(self) -> bool:
        """断开连接"""
        pass
    
    @abstractmethod
    async def get_state(self) -> HardwareState:
        """获取硬件状态"""
        pass
    
    @abstractmethod
    async def enable(self) -> bool:
        """启用硬件"""
        pass
    
    @abstractmethod
    async def disable(self) -> bool:
        """禁用硬件"""
        pass

class ArmInterface(HardwareInterface):
    """机械臂接口"""
    
    @abstractmethod
    async def move_joints(self, joints: List[float], **kwargs) -> bool:
        """关节空间运动"""
        pass
    
    @abstractmethod
    async def move_pose(self, pose: List[float], **kwargs) -> bool:
        """笛卡尔空间运动"""
        pass
    
    @abstractmethod
    async def get_current_joints(self) -> List[float]:
        """获取当前关节角度"""
        pass
    
    @abstractmethod
    async def get_current_pose(self) -> List[float]:
        """获取当前末端位姿"""
        pass
    
    @abstractmethod
    async def get_ik(self, pose: List[float], ref_joints: List[float]) -> List[float]:
        """逆运动学求解"""
        pass

class BaseInterface(HardwareInterface):
    """底盘接口"""
    
    @abstractmethod
    async def navigate_to(self, target: str, **kwargs) -> bool:
        """导航到目标"""
        pass
    
    @abstractmethod
    async def move_velocity(self, vx: float, vy: float, omega: float) -> bool:
        """速度控制"""
        pass
    
    @abstractmethod
    async def get_pose(self) -> List[float]:
        """获取当前位姿"""
        pass

class GripperInterface(HardwareInterface):
    """夹爪接口"""
    
    @abstractmethod
    async def open(self) -> bool:
        """打开夹爪"""
        pass
    
    @abstractmethod
    async def close(self, force: float = 100) -> bool:
        """关闭夹爪"""
        pass
    
    @abstractmethod
    async def set_position(self, position: float) -> bool:
        """设置位置"""
        pass
    
    @abstractmethod
    async def get_position(self) -> float:
        """获取当前位置"""
        pass
```

### 5.2 统一RobotAPI

```python
# voice/hardware/robot_api.py

class UnifiedRobotAPI:
    """统一机器人API"""
    
    def __init__(self, hardware_config: dict):
        self.config = hardware_config
        self._interfaces: Dict[str, HardwareInterface] = {}
        self._initialize_hardware()
    
    def _initialize_hardware(self):
        """根据配置初始化硬件"""
        # 初始化机械臂
        for arm_config in self.config.get("arms", []):
            arm_interface = self._create_arm_interface(arm_config)
            self._interfaces[arm_config["name"]] = arm_interface
        
        # 初始化底盘
        if "base" in self.config:
            base_interface = self._create_base_interface(self.config["base"])
            self._interfaces[self.config["base"]["name"]] = base_interface
        
        # 初始化夹爪
        for gripper_config in self.config.get("grippers", []):
            gripper_interface = self._create_gripper_interface(gripper_config)
            self._interfaces[gripper_config["name"]] = gripper_interface
    
    def _create_arm_interface(self, config: dict) -> ArmInterface:
        """创建机械臂接口（工厂方法）"""
        arm_type = config["type"]
        
        if arm_type.startswith("jaka"):
            return JAKAArmInterface(config)
        elif arm_type == "ur":
            return URArmInterface(config)
        elif arm_type == "franka":
            return FrankaArmInterface(config)
        else:
            raise ValueError(f"不支持的机械臂类型: {arm_type}")
    
    # 属性访问器
    @property
    def arms(self) -> Dict[str, ArmInterface]:
        """获取所有机械臂接口"""
        return {k: v for k, v in self._interfaces.items() 
                if isinstance(v, ArmInterface)}
    
    @property
    def base(self) -> Optional[BaseInterface]:
        """获取底盘接口"""
        for interface in self._interfaces.values():
            if isinstance(interface, BaseInterface):
                return interface
        return None
    
    @property
    def grippers(self) -> Dict[str, GripperInterface]:
        """获取所有夹爪接口"""
        return {k: v for k, v in self._interfaces.items() 
                if isinstance(v, GripperInterface)}
    
    # 便捷方法
    def get_arm(self, name: str = None) -> ArmInterface:
        """获取指定机械臂（默认第一个）"""
        arms = self.arms
        if name:
            return arms[name]
        return list(arms.values())[0] if arms else None
    
    async def connect_all(self) -> bool:
        """连接所有硬件"""
        results = []
        for interface in self._interfaces.values():
            results.append(await interface.connect())
        return all(results)
    
    async def get_all_states(self) -> Dict[str, HardwareState]:
        """获取所有硬件状态"""
        states = {}
        for name, interface in self._interfaces.items():
            states[name] = await interface.get_state()
        return states
```

### 7.2 与现有研究的对比

> ⚠️ **重要说明**：下表为**作者根据论文内容整理的方法论对比**，非实验对比数据。不同方法的适用场景、技术路线、评估标准均不同，此对比仅展示**设计思路的差异**。

| 方面 | Open X-Embodiment | Octo | 我们的方案 |
|------|-------------------|------|-----------|
| 硬件描述 | 隐式（数据驱动） | 隐式 | **显式（YAML配置）** |
| 代码生成 | 无 | 无 | **LLM驱动** |
| 接入成本 | 需要大量数据 | 需要微调 | **配置即可** |
| 可解释性 | 低 | 低 | **高（代码可见）** |

**各方法核心差异**：
- **Open X-Embodiment**：通过大规模跨机器人数据训练，实现隐式的硬件泛化
- **Octo**：通过少量微调适配新硬件，仍需数据收集
- **我们的方案**：通过显式配置+代码生成，无需训练即可适配
