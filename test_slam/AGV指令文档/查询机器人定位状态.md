**查询机器人定位状态**

**请求**

编号: 1021 (0x03FD)

名称: robot_status_reloc_req

描述: 查询机器人当前的定位状态

JSON 数据区: 无

**请求示例**

Plaintext  
5A 01 00 01 00 00 00 00 03 FD 00 00 00 00 00 00

**响应**

编号: 11021 (0x2B0D)

名称: robot_status_reloc_res

描述: 查询机器人当前的定位状态的响应

JSON 数据区: 见下表

如果响应数据区的 JSON Object 中有 API 中未提到的 key-value 键值对，可以直接忽略，无须关注其含义，下文不再赘述

|     |     |     |     |
| --- | --- | --- | --- |
| 字段名 | 类型  | 描述  | 可缺省 |
| reloc_status | number | 0 = RELOC_INIT(重定位初始化中)<br><br>1 = RELOC_SUCCESS(重定位成功)<br><br>2 = RELOC_RELOCING(正在重定位) | 是   |
| ret_code | number | API 错误码 | 是   |
| create_on | string | API 上传时间戳 | 是   |
| err_msg | string | 错误信息 | 是   |

注意:

定位状态 (1021) 用于指示当前机器人定位状态是否正确，对于 3.4.6.1800 之前的版本，定位状态如果为 3 = COMPLETED, 说明重定位已经完成，但是用户没有确认，此时需要通过 (2003) 来进行确认或重新进行定位。用户确认定位正确后，定位状态会变为 SUCCESS

**响应示例**

JSON  
{  
"reloc_status": 1  
}
