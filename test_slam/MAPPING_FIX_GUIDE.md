# AGV建图流程修正说明

## 问题描述

原始的 `stage1_mapping` 方法在停止建图后存在问题：
1. **未正确获取地图数据**：停止建图后直接等待5秒就结束，没有获取建图结果
2. **地图载入验证缺失**：没有检查当前载入的地图是否是刚才扫描生成的地图
3. **地图上传功能缺失**：如果当前载入的不是新建图，没有自动上传载入新地图

## 修正方案

根据AGV指令文档《查询机器人扫图状态》，修正后的流程如下：

### 核心修正点

#### 1. 轮询获取地图数据
```python
# 修正前（错误）
print("等待建图数据保存...")
time.sleep(5)  # 只是简单等待

# 修正后（正确）
new_map_data = self._wait_for_map_data(timeout=60)
```

**原因**：文档明确指出"停止建图后，不会立即返回数据（数据有可能为空），请继续轮询直至返回非空数据"

#### 2. 地图载入验证和自动上传
```python
# 新增功能
if not self._ensure_new_map_loaded(new_map_data):
    print("地图载入验证失败")
    return False
```

**功能**：
- 检查当前载入地图是否为新建图（通过MD5比较）
- 如果不是，自动上传并载入新地图
- 验证上传载入是否成功

#### 2. `_ensure_new_map_loaded()`
```python

    """确保新建图的地图已经载入"""
```

**功能**：
- 获取当前载入地图信息（API 1300）
- 计算新地图和当前地图的MD5
- 如果不匹配，上传新地图（API 2025）
- 验证上传是否成功

## 使用示例

### 修正前的问题
```python
# 原来的代码（有问题）
if not self.agv.stop_slam():
    print("停止建图失败")
    return False

print("等待建图数据保存...")
time.sleep(5)  # 只是等待，没有获取数据
print("建图完成")
```

### 修正后的正确流程
```python
# 修正后的代码
if not self.agv.stop_slam():
    print("停止建图失败")
    return False

# 轮询获取地图数据
new_map_data = self._wait_for_map_data(timeout=60)
if new_map_data is None:
    print("获取建图数据失败")
    return False

# 检查并载入新地图
if not self._ensure_new_map_loaded(new_map_data):
    print("地图载入验证失败")
    return False

print("建图完成，新地图已载入")
```

## 关键改进

### 1. 数据完整性
- **修正前**：可能丢失建图数据
- **修正后**：确保获取完整地图数据

### 2. 地图一致性
- **修正前**：不知道当前载入的是什么地图
- **修正后**：确保载入的是最新建图

### 3. 自动化程度
- **修正前**：需要手动管理地图
- **修正后**：自动上传载入新地图

### 4. 错误处理
- **修正前**：简单等待，无错误检测
- **修正后**：完整的超时和错误处理

## 技术要点

### API使用规范
1. **查询扫图状态**：必须设置 `return_resultmap=True` 才能获取地图数据
2. **轮询机制**：必须持续查询直到 `resultmap` 非空
3. **地图上传**：直接发送JSON格式地图数据，不是包装在JSON对象中

### MD5验证机制
- 新地图MD5 = MD5(map_data.encode('utf-8'))
- 当前地图MD5通过API 1300获取
- 只有MD5不匹配才上传新地图

### 地图命名规则
- 格式：`slam_map_YYYYMMDD_HHMMSS`
- 示例：`slam_map_20241201_143022`
